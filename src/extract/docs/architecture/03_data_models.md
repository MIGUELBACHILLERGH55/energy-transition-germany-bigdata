PlanItem → Purpose & Fields ^eLqsox25

 Represents a single unit of work planned by the extractor,
  before any data is actually fetched.
  It defines WHAT to fetch (or read), not how or when to parse it.
  No network or file I/O happens here — it is just a plan ^M2VqUXSJ

Typical fields:
  - dataset (DatasetSpec) → dataset this work item belongs to
  - window (tuple[str, str] | None) → time interval or logical chunk boundary
  - endpoint (str, opt) → API relative path (if access = "api")
  - params (dict, opt) → resolved query parameters for the API
  - file_path (str, opt) → resolved file path (if access = "file")
  - tags (dict, opt) → extra labels (run_id, attempt, etc.) ^99SMakyo

Output used by:
  DownloaderStrategy (via fetch(plan_item)) ^2IYd2ous

Notes:
  - Produced by the extractor’s plan() method.
  - No network or file I/O occurs here — this is purely a planning object.
  - Later consumed by fetch(), which executes the retrieval based on this plan. ^pF9NWwym

RawResponse → Purpose & Fields ^r0P9AClA

  Wraps the raw payload returned by fetch() before any parsing.
  Represents data exactly as received (e.g. HTTP bytes, local file 
  content).
  No transformation or validation is applied at this stage ^9rZjWe7G

Typical fields:
  - content (bytes | str | IOBase) → the raw body or file stream
  - status (int | str | None) → HTTP status code or equivalent
  - dataset_id (str) → which dataset this response belongs to
  - fetched_at (datetime) → timestamp of when data was retrieved
  - source_meta (dict, opt) → source-specific metadata
  - meta (dict, opt) → extra info (headers, file size, checksum, etc.) ^bhktLwbZ

Notes:
  - Produced by fetch(), consumed by parse().
  - Still represents raw, unprocessed data — not cleaned or validated.
  - No business logic, normalization, or schema enforcement applied yet. ^pJRyQVqN

ParsedBatch → Purpose & Fields ^Ae6RdsZI

 Represents minimally structured, validated data produced by parse().
  It is ready to be persisted into the landing layer 
(no heavy transformations) ^vSI9S5ru

Typical fields:
  - records (list[dict] | DataFrame) → decoded rows, minimally cleaned
  - dataset_id (str) → traceability (matches metadata.dataset_id)
  - window (tuple[start, end] | None) → time coverage for this batch
  - rows (int) → record count after cleaning
  - meta (dict) → dtypes, columns, tz_info, notes ^avur5Z7V

Notes:
  - Represents clean-enough data for landing: timestamps and types parsed, columns consistent.
  - No business-heavy normalization (reserved for Bronze/Silver layers).
  - Idempotent when combined with atomic file naming in persist_landing(). ^dg6WYR4A

Output used by:
  StorageClient (via persist_landing()) ^sloPX38B
